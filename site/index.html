Traced case study, run 01

The Telephone Game

Asish Singh · July 2026 · every figure generated from the real 18 signal trace

This is a study of one real build task, traced signal by signal, to see how an instruction changes as it moves from a person's intent, through an orchestrating AI (Claude), to the coding subagents (Codex) that actually write the code. Every message between the layers was logged and every layer's understanding of the task was captured and compared back to the original ask. The short version, most of the instruction survived untouched, two small pieces of it changed at the very first handoff, and three real defects were caught and fixed by the review loop before anything shipped.

The finding, up front

  • 7 of 9 canonical requirements from the original ask survived every hop intact, from the human's words to the merged code.
  • Both requirements that changed did so at the very first hop, when Claude turned the intent into instructions for the coding subagents. Nothing drifted further after that point, and nothing drifted anywhere else.
  • Three correction loops between Claude and the coding subagent each caught a real defect that the subagent's own tests had missed, and each one ended in an explicit acceptance before the work moved on.

MethodHow the trace works

The human (Asish) wrote one intent message, once, without coaching. Claude read it and decomposed it into instructions for two coding subagents, codex-1 and codex-2. Codex-1 built the extraction engine, codex-2 built the report writer, and Claude reviewed both, sending corrections back when something was wrong. Eighteen signals were logged this way, timestamped and typed, from the first intent message to the final handoff back to the human. Separately, a neutral model with no part in the build read a fixed six question snapshot at each layer (the intent, the decomposition, each subagent's own reading of its instruction, and the finished code) and classified every one of the nine original requirements as preserved, mutated, dropped, or invented at each layer. Nothing here is hand drawn, every figure below comes from that trace and that classification.

FindingsThree correction stories

The review loop between Claude and codex-1 fired three times. Each time, Claude's acceptance run found something the subagent's own tests had passed anyway, sent a correction, and got a fix back before accepting the work.

seq 5 to 7, keyframes lost to a broken CSS parser

Zero keyframes, despite forty animations in use

Codex-1 reported four out of four unit tests passing. Claude's own acceptance run on the real Shopify page told a different story, zero CSS keyframes were captured even though the page used forty named animations, and the section outline came back as a flat two hundred and twenty item list instead of a readable top level structure. The correction asked for a token aware CSS parser and a top level outline with eight to twenty five entries. Codex-1 rewrote the parser, all five tests passed, and it came back for another look.

seq 8 to 10, an outline regression on the simple page

Shopify fixed, example.com went empty

The rewritten parser passed on Shopify, fourteen keyframes, no unmatched entries, seventeen sections. But the second, simpler test page, example.com, now returned zero of one sections, an outline regression the new logic had introduced while fixing the first problem. The correction asked for a fallback, if the strict outline logic finds nothing, fall back to the best available candidates rather than returning empty. Codex-1 added the fallback with its own unit test, six of six tests passed.

seq 14 to 16, non visual markup leaking into the color report

head, meta, and title showing up as color evidence

By this point codex-2's report writer was done, ten of ten tests passing, and the end to end run looked exhaustive and correct. But Claude's review of the color evidence in the generated report noticed entries coming from elements that are never rendered, head, meta, and title tags. The correction went back to codex-1, exclude non rendered elements (head, meta, title, script, style, link) from computed style inspection. The fix landed, eleven of eleven tests passed, and a fresh capture of Shopify confirmed no non visual selectors remained in the color evidence.

HonestyLimitations

  • This is one run of one build task. Nothing here claims to generalize to other tasks, other supervising models, or other coding agents.
  • The requirement classification was done by a single neutral model reading fixed snapshots, not by multiple independent raters, so the classification itself carries whatever judgment that one model brought to borderline calls, such as reading R8's "inline-encoded elements" as inline SVGs specifically.
  • The trace captures signals between layers, not every private reasoning step inside Claude or inside either Codex subagent, so drift that never surfaced as a logged signal would not appear here.
  • If a second run showed more drift, or drift recovered less cleanly, that would be the finding for that run, this page only reports what run 01 actually shows.
