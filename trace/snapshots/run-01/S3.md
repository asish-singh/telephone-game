# 1. What is being built, in one sentence?

A Node.js CLI tool named `designdna` that loads a URL in headless Chromium and captures the "visual design DNA" of the rendered page (colors, typography, spacing, breakpoints, animations, backgrounds, inline SVGs, and page-outline sections) as structured JSON and/or a Markdown design report ("DESIGN.md" / capture-schema.md).

# 2. List every required behavior (numbered, one per line, verbatim quotes where possible).

1. Package description: "Capture the visual design DNA of a rendered web page." (package.json)
2. Exposes a bin named `designdna` at `bin/designdna.js`; `npm run capture` runs it, `npm test` runs `node --test`, `npm run smoke` runs `scripts/smoke.js`.
3. CLI usage: "Usage: node bin/designdna.js <url> [-o <DESIGN.md>] [--json <capture.json>]" (bin/designdna.js).
4. CLI accepts a single positional URL argument, `--json <path>`/`--json=<path>`, `-o|--output <path>`/`--output=<path>`, and `--help|-h`.
5. Requires a URL, and requires that at least one of `--json` or `--output` be supplied, else throws the usage error.
6. Validates the URL is parseable and restricts protocol to `http:` or `https:` ("Only http:// and https:// URLs are supported.").
7. Writes JSON output as `${JSON.stringify(capture, null, 2)}\n` and/or the rendered Markdown report to the requested paths, creating parent directories as needed (`mkdir(dirname(path), { recursive: true })`).
8. After running, logs a one-line summary: "Captured N colors, M keyframes, and K sections."
9. `extractDesign(url)` launches headless Chromium via Playwright with a fixed viewport of 1440×900 CSS pixels, device scale factor 1, light color scheme, `en-US` locale, and `reducedMotion: "no-preference"` ("normal (not reduced) motion", per docs/capture-schema.md).
10. `extractDesign` rejects non-http(s) URLs before browser launch ("extractDesign only supports http:// and https:// URLs").
11. Instruments the page before scripts run to record `IntersectionObserver.observe` targets and canvas `getContext` calls/types (for scroll-trigger and hero-canvas detection).
12. Navigates with `waitUntil: "domcontentloaded"` and a 60s timeout, then waits for network-idle (10s timeout, non-fatal), an extra 1.5s timeout, and web-font readiness (`document.fonts.ready` raced against an 8s timeout).
13. Captures `meta`: final URL, requested URL, page title, viewport, ISO-8601 UTC timestamp, document language/scrollWidth/scrollHeight/contentType, and navigation status/statusText.
14. Captures `colors`: descending-by-count list of normalized computed colors with role counts (text/background/border/accent), contributing CSS property counts, and up to 6 example selectors; reads color-bearing properties, fills, strokes, shadows, decoration, border images, and background images; retains transparent colors.
15. Captures `typography`: complete computed `font-family` stacks and their variants (size, weight, line-height, letter-spacing, font-style) with counts, example element, and whitespace-normalized sample text truncated to 160 characters; collected from visible elements with rendered text, form values/placeholders, image alt text, and generated content.
16. Captures `spacing`: common margin/padding values, ranked margins, paddings, container max-widths (omitting `none`/`0px`), border radii (omitting `0px`), and box-shadows (omitting `none`); margin/padding counts include every longhand side including zero.
17. Captures `breakpoints`: every discovered `@media` occurrence (including nested), a summary grouped by exact condition string, and a stylesheet inventory record per traversed sheet (source, origin, cross-origin flag, status, method, status code, bytes, error). `approximatePixels` converts `em`/`rem` at 16px, else `null`.
18. External stylesheets are recovered first from navigation/network responses, then fetched via the browser context; reachable `@import` URLs are traversed recursively; raw sheets over 20 MiB are skipped and traversal stops beyond 500 sheets, both with notes; CSSOM-accessible rules are retained as a fallback when raw fetch fails.
19. Captures `animations`: every `@keyframes` occurrence (standard or vendor-prefixed, duplicate names across sheets kept separate) with verbatim/serialized `cssText`, computed active uses, rule-based (e.g. hover) uses, best-effort `likelyTriggers` (load/hover/focus/active/scroll) with trigger evidence, unmatched animation uses, and IntersectionObserver hints; also captures non-zero-duration transitions per element/pseudo-element.
20. Captures `backgrounds`: one background record per emitted outline section (color, image, extracted `url(...)` list, size/position/repeat/attachment/origin/clip/blend-mode/mix-blend-mode/filter/backdrop-filter), plus hero video/canvas media detected in hero or above-the-fold sections (video: source URLs, autoplay/muted/loop/playsInline/poster; canvas: bitmap size and observed context types, marking `webgl-canvas` when webgl/webgl2 was requested).
21. Captures `inlineSvgs`: every `<svg>` in the document, with `kind: "verbatim"` (including markup) for items under a 4096-byte UTF-8 serialized-`outerHTML` threshold, otherwise `kind: "summary"` (metadata only); infers `role` when not explicit (decorative/image/icon/graphic).
22. Captures `sections`: a top-level page outline in document order (landmarks, primary/header navigation, top-level section/article/ARIA regions, explicit hero/masthead blocks, large full-width direct children of body/main), excluding generic wrappers, nested cards, secondary navigation, hidden/zero-size elements; each item has kind, selector, tag, id, classes, role, dimensions, document top offset, up to 3 headings, key computed styles, structural counts (direct children, headings, links, buttons, images), and a generated description string; falls back to the largest visible candidate(s) when no major candidate passes filtering.
23. Every major extraction field has a `notes: string[]`; a failed sub-extraction produces an empty value plus a note instead of aborting the whole capture (enforced via a `safely()` wrapper around each extraction step).
24. Iframe presence is detected and recorded in notes; iframe contents are not traversed.
25. `renderDesignMd(capture)` renders a schema 1.0.0 capture into deterministic Markdown: header/theme summary, design tokens (colors, typography, spacing, breakpoints), layout system (sections), animations, backgrounds/effects, a replication checklist, and a fidelity-notes section that walks the whole capture object collecting every `notes`/`error` field with its path.
26. `renderDesignMd` throws a `TypeError` if not given a capture object, and a `RangeError` if `capture.schemaVersion` is present and not equal to `"1.0.0"` ("Unsupported design capture schema version: ...").
27. `scripts/smoke.js` runs `extractDesign` against two fixed URLs (`https://example.com/` and `https://www.shopify.com/editions/winter2026`), writes each result JSON to `captures/<name>.json`, logs a summary line per target, and sets a non-zero exit code if any target fails.
28. Consumers of the JSON schema "must check `schemaVersion` before decoding"; new optional properties may be added within a schema version but renames/removals/type changes require a new version (docs/capture-schema.md).

# 3. List every explicit constraint (style, tools, limits).

1. Runtime: Node.js `>=20`, ESM (`"type": "module"`).
2. Single runtime dependency: `playwright` (`^1.61.1`).
3. Browser engine: Chromium only (`chromium.launch`), headless.
4. Fixed viewport 1440×900, device scale factor 1, light color scheme, `en-US` locale, non-reduced motion — not configurable via CLI flags.
5. URL protocol restricted to `http:`/`https:` in both the CLI argument parser and `extractDesign`.
6. Output format is UTF-8 JSON; schema version is `"1.0.0"`; consumers must check `schemaVersion`.
7. `MAX_EXAMPLE_SELECTORS = 6`: at most six example selectors stored per aggregated entry (colors, spacing values, etc.).
8. `MAX_STYLESHEETS = 500`: stylesheet/`@import` traversal capped at 500 sources.
9. `MAX_STYLESHEET_BYTES = 20 * 1024 * 1024` (20 MiB): raw stylesheets larger than this are skipped.
10. Inline-SVG verbatim-markup threshold: 4096 bytes (`InlineSvgs.thresholdBytes`); at/above this, only a metadata summary is kept (no markup).
11. Navigation timeout 60s (`waitUntil: "domcontentloaded"`); network-idle wait timeout 10s; fixed extra settle wait 1.5s; web-fonts-ready race timeout 8s; stylesheet fetch timeout 30s.
12. Section outline cap: `maxSections = 50` (from `collectPageOutline` default and its call site).
13. Error messages are cleaned of ANSI escape codes and truncated to 4000 characters (`errorMessage`).
14. Every non-fatal extraction failure must be caught (`safely()`), logged to that field's `notes`, and replaced with a fallback value rather than aborting the whole capture.
15. `renderDesignMd` must be deterministic and must not mutate its input capture (asserted directly in test/report.test.js).
16. Selectors are documented as "stable best-effort hints, not guaranteed to remain queryable after the page mutates" — not a strict contract.
17. Counts throughout are computed-style/property occurrence counts, explicitly not pixel-area measurements.
18. Open-shadow-root selectors use `host-selector >>> inner-selector` notation; closed shadow roots are implicitly unreachable (not stated but implied by using `element.shadowRoot`).

# 4. List the edge cases that must be handled.

1. Chromium launch failure (wrapped and rethrown as "Chromium launch failed: ...").
2. Navigation failure/timeout (caught, recorded as a `meta.notes` entry, extraction continues).
3. Missing main-resource navigation response (`navigation: { status: null, statusText: "Navigation returned no main-resource response" }`).
4. Network-idle wait timing out (non-fatal, logged).
5. Web fonts never becoming ready within 8s (raced against a timeout so it can't hang).
6. Any single DOM element throwing during computed-style inspection (caught per-element; failure count reported in a shared note across colors/typography/spacing/animations fields, e.g. "N DOM elements could not be inspected and were skipped.").
7. Elements with `display:none` or zero-size bounding rect are excluded from visibility-gated data (typography/spacing) but colors/motion are still read for such elements per the test ("keeps display-none elements" for color inspection specifically).
8. `currentColor` tokens resolved against the element's own computed `color`.
9. Un-parseable/invalid color tokens (probe returns empty style) are skipped.
10. No major outline candidates pass filtering: falls back to up to 5 (or `min(5, maxSections)`) largest visible candidates, with a note explaining the fallback.
11. Zero visible candidates at all: outline/backgrounds/hero media all empty, with an explanatory note.
12. Malformed/unterminated CSS blocks (unterminated `{`), malformed/unnamed `@keyframes` — logged as notes, parsing continues for the rest of the stylesheet, and a token-aware scan protects real keyframes from being hidden by a malformed surrounding block (explicitly tested with string/comment/url decoys and a truncated trailing block).
13. Duplicate `@keyframes` names defined in different stylesheets are kept as separate occurrences, not merged.
14. Vendor-prefixed `@keyframes` (e.g. `@-webkit-keyframes`) are recognized and their prefix recorded.
15. `data:`-URL stylesheets are decoded (base64 or percent-encoded) rather than fetched; `blob:` stylesheet URLs throw ("blob: stylesheet cannot be fetched outside the page") and are recorded as a failed/skipped stylesheet.
16. Cross-origin stylesheet fetch failures, non-OK HTTP status, or CSSOM access errors are recorded per-sheet as `status: "failed"` with an error message, while still allowing accessible CSSOM rules as a fallback.
17. Stylesheets exceeding the 20 MiB byte limit are recorded as `status: "skipped"` with a note, not parsed.
18. Traversal discovering more than 500 stylesheets/imports stops and logs a note with the true discovered count.
19. Unresolvable or unsupported `@import` URL schemes are skipped with a note (only `http:`, `https:`, `data:`, `blob:` schemes considered, and `blob:` ultimately fails at fetch time).
20. Computed animation names with no matching `@keyframes` definition anywhere are reported under `unmatchedAnimationUses` rather than silently dropped.
21. Media query width extraction handles both legacy (`min-width`/`max-width`) and modern range syntax (`width < 1200px`, `600px <= width`), including `em`/`rem`/`vw` units, converting `em`/`rem` to approximate pixels at 16px and leaving other units as `null`.
22. Shadow DOM: open shadow roots are traversed recursively for elements, stylesheets, and adopted stylesheets; selectors for elements inside a shadow root are prefixed with the host selector via `>>>` notation.
23. Elements/sections whose selector collides with siblings get an `:nth-of-type(n)` disambiguator; unresolvable/ambiguous selectors fall back to a longer structural path.
24. Iframes are detected (count captured) but their internal documents are not traversed; recorded as a note.
25. `renderDesignMd` given an empty object `{}` must still render every section using "No ... were captured." placeholder text instead of throwing.
26. `renderDesignMd` given a capture with `schemaVersion` other than `"1.0.0"` throws a `RangeError`; a capture with no `schemaVersion` at all is still accepted.
27. Table/markdown-cell values escape HTML-sensitive and Markdown-sensitive characters (`|`, backticks, `*`, `_`, `[`, `]`, newlines) so captured content can't corrupt the rendered Markdown; code fences pick a longer backtick run than any backtick sequence already present in the fenced content.
28. `smoke.js`: if one target fails during capture, the failure is logged with the target name and first line of the error, the other target still runs, and the process exits non-zero only if at least one target failed.
29. CLI argument parsing: rejects unknown arguments, missing required values for `--json`/`--output` (including a following token that itself looks like a flag), and invalid URLs, each with a specific error message.

# 5. What is explicitly out of scope?

1. Only Chromium is used; no explicit support for other engines (Firefox/WebKit) despite Playwright supporting them.
2. Iframe document contents are explicitly not traversed/captured, only their presence is noted.
3. Closed shadow roots are not addressed (only `element.shadowRoot`, which is null for closed roots, is used).
4. `blob:` stylesheet URLs cannot be fetched/read outside the page context and are excluded.
5. Non-`http`/`https` URLs are rejected outright, both at the CLI layer and inside `extractDesign`.
6. Viewport size, device scale factor, color scheme, locale, and motion preference are fixed constants; the CLI does not expose flags to change them.
7. Pixel-area/visual-coverage measurement is explicitly disclaimed — counts are computed-style occurrence counts, not pixel-area measurements.
8. Selector stability across page mutation is explicitly not guaranteed ("stable best-effort hints, not guaranteed to remain queryable after the page mutates").
9. Trigger detection (hover/focus/active/scroll/load) is explicitly "best-effort evidence, not guaranteed behavior."
10. Schema evolution: renames, removals, or type changes to schema 1.0.0 fields are out of scope for a patch/minor bump — they "require a new version."
11. `captures/` and `examples/` directories are not part of the tool's source/spec surface (excluded from this review per task instructions, and functionally: `captures/` is smoke-test/test-fixture output, `examples/` holds a stale-check demonstration artifact referenced only by a test).

# 6. How will success be checked?

1. `npm test` runs `node --test`, which executes `test/extract.test.js` and `test/report.test.js`.
2. `test/extract.test.js` verifies (via the exported `__internal` object): CSS traversal correctness for keyframes/media/state/timeline rules including `@import`, vendor prefixes, and nested `@supports`; robustness of keyframe extraction against minified CSS, string/comment decoys, and a truncated trailing block; correct legacy and range-syntax media-width parsing; correct association of computed animation uses with keyframes and inferred likely triggers (hover, scroll via `animation-timeline`); that `extractDesign` rejects non-http(s) URLs (`file://`) before any browser work; that computed-style collection skips non-rendered elements (head/script/style/link/template/title) while still inspecting display:none elements, using injected fake DOM globals; that the page outline correctly falls back to the single visible section on a minimal page and records the fallback note.
3. `test/report.test.js` verifies: `renderDesignMd` is deterministic (same input twice produces identical output) and does not mutate the input capture object; a full fixture capture renders expected content including the header, hex-converted color, verbatim keyframe/media-query/SVG text, deduplicated transition occurrence counts, stylesheet error paths, and root-level notes; an empty `{}` capture renders "not captured" placeholder text for every section without throwing; an unsupported `schemaVersion` (e.g. `"2.0.0"`) throws; rendering real captured fixtures (`captures/example.json`, `captures/shopify-winter2026.json`) produces non-empty reports that include every keyframe name and its verbatim CSS text, and for the Shopify fixture specifically, the rendered report must exactly equal the checked-in `examples/DESIGN-shopify.md` demonstration artifact (a staleness check).
4. `npm run smoke` (`scripts/smoke.js`) is a live, network-dependent smoke test: it runs `extractDesign` against two real URLs, writes their captures to `captures/`, prints a summary line per target, and exits non-zero if any target's capture threw.
5. Manual/CLI-level check: running `node bin/designdna.js <url> -o DESIGN.md --json capture.json` should print "Wrote <path>" for each requested output and the final "Captured N colors, M keyframes, and K sections." summary line, with a non-zero exit code and a `designdna: <message>` error line on failure.
