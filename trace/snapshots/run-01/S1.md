# 1. What is being built, in one sentence?

A Node.js CLI tool called "designdna" with two parts: an extraction engine (src/extract.js, bin/designdna.js) that renders a URL in headless Chromium via Playwright and produces a structured JSON "design capture" of its visual design, and a report writer (src/report.js) that turns that capture into an exhaustive, agent readable DESIGN.md replication guide.

# 2. List every required behavior (numbered, one per line, verbatim quotes where possible)

1. "package.json with playwright as dependency, run `npx playwright install chromium` so it works."
2. "src/extract.js exporting `extractDesign(url) -> capture object`, and bin/designdna.js as CLI: `node bin/designdna.js <url> --json capture.json`."
3. The capture object must include a `meta` field: "url, title, viewport used, timestamp"
4. The capture object must include `colors`: "frequency ranked list of all colors from computed styles, each with role hints (text, background, border, accent) and example selectors"
5. The capture object must include `typography`: "every font family in use, and for each a list of (size, weight, lineHeight, letterSpacing, example element/selector, sample text)"
6. The capture object must include `spacing`: "common margin/padding values ranked, container max widths, border radii, box shadows"
7. The capture object must include `breakpoints`: "from all stylesheets' media queries"
8. The capture object must include `animations`: "EVERY @keyframes rule found in all reachable stylesheets (verbatim CSS text), plus all transitions from computed styles (selector, properties, duration, easing), plus which elements use each keyframe animation." Also "Note likely triggers (load, hover via :hover rules, scroll via IntersectionObserver hints or animation-timeline) where detectable."
9. The capture object must include `backgrounds`: "for major sections, computed background (colors, gradients verbatim, image URLs), blend modes, filters, backdrop filters; detect video/canvas/WebGL hero elements"
10. The capture object must include `inlineSvgs`: "inline <svg> elements under 4KB verbatim, larger ones summarized (id/class, size, role)"
11. The capture object must include `sections`: "top level page outline (header, nav, hero, each major section, footer) with tag, classes, approximate height, key computed styles, and a one line structural description"
12. "Handle cross origin stylesheets gracefully (fetch their text when possible, skip with a note when not)."
13. "Write docs/capture-schema.md documenting the exact JSON shape you emit (another developer codes against this)."
14. "Test against https://www.shopify.com/editions/winter2026 and also a simple site like https://example.com."
15. "Print a short report of what the shopify capture contained (counts of colors, keyframes, sections) at the end."
16. "src/report.js exporting `renderDesignMd(capture) -> string`, producing an exhaustive, agent readable DESIGN.md."
17. "Wire it into bin/designdna.js so `node bin/designdna.js <url> -o DESIGN.md` runs extract then report (keep the existing --json flag working; both flags may be combined)."
18. DESIGN.md section 1, Header: "site title, URL, capture date, one paragraph theme summary derived from the data (dominant palette, typographic character, layout feel). Derived deterministically from the capture, no AI calls, no invented adjectives beyond mechanical mappings (e.g. dark background ratio -> \"dark theme\")."
19. DESIGN.md section 2, Design tokens: "Colors as a table (hex, role hints, usage frequency, example selectors). Typography as a table per font family. Spacing scale, radii, shadows, container widths, breakpoints."
20. DESIGN.md section 3, Layout system: "Section outline as a numbered walkthrough, each entry with its structural description and key computed styles as a small CSS block."
21. DESIGN.md section 4, Animations ("the flagship chapter"): "EVERY keyframe verbatim in a css code block, followed by a plain english description of what it does (properties animated, duration, easing, likely trigger) and which elements use it. Then a transitions table (selector, properties, duration, easing) deduplicated sensibly."
22. DESIGN.md section 5, Backgrounds and effects: "Gradients verbatim, background images with URLs, blend modes, filters, video/canvas heroes, notable inline SVGs in svg code blocks."
23. DESIGN.md section 6, Replication checklist: "a numbered list an AI agent can follow to rebuild the look (load fonts X, define tokens, build sections in order, add keyframes, wire triggers)."
24. DESIGN.md section 7, Fidelity notes: "surface every note/error recorded in the capture so the reader knows what the capture could not see."
25. "Handle missing/empty fields gracefully (omit sections with a one line note rather than crashing)."
26. "Escape markdown safely."
27. "Add unit tests (test/report.test.js) using a small fixture capture plus the two real captures (assert it renders without throwing and includes every keyframe name)."
28. "Run npm test."
29. "Generate DESIGN-shopify.md from captures/shopify-winter2026.json into examples/ as a demonstration artifact."
30. "Code strictly against the schema doc and verify against the real captures."

# 3. List every explicit constraint (style, tools, limits)

1. "plain Node.js (ES modules), no TypeScript, no framework beyond Playwright"
2. "Keep it robust; a failed sub extraction must not crash the run, record an error note in that field instead."
3. "Commit nothing; just leave working code." (for the extraction engine task)
4. "existing Node.js project, do not modify src/extract.js"
5. "deterministic pure function, plain Node ES modules, no dependencies" (for report.js)
6. "no AI calls, no invented adjectives beyond mechanical mappings" (for the theme summary)
7. Report writer must "code strictly against the schema doc" (docs/capture-schema.md) and verify against the real captures.

# 4. List the edge cases that must be handled

1. Cross origin stylesheets that cannot be fetched: "fetch their text when possible, skip with a note when not."
2. A failed sub extraction (of any field) must not crash the run; record an error note in that field instead.
3. Large inline SVGs (4KB or over): summarized (id/class, size, role) rather than included verbatim.
4. Missing or empty fields in the capture when rendering DESIGN.md: "omit sections with a one line note rather than crashing."
5. Markdown content that needs safe escaping.
6. Notes/errors recorded in the capture must be surfaced in the Fidelity notes section so readers know what the capture could not see.

# 5. What is explicitly out of scope?

1. TypeScript.
2. Any framework beyond Playwright (for the extraction engine).
3. Any dependencies for src/report.js ("no dependencies").
4. AI calls or invented adjectives in the theme summary (only mechanical/deterministic mappings from the data).
5. Modifying src/extract.js when building the report writer ("do not modify src/extract.js").
6. Committing anything during the extraction engine task ("Commit nothing; just leave working code").

# 6. How will success be checked?

1. Test against https://www.shopify.com/editions/winter2026 and https://example.com, printing a short report of what the shopify capture contained (counts of colors, keyframes, sections).
2. Unit tests in test/report.test.js using a small fixture capture plus the two real captures (captures/example.json and captures/shopify-winter2026.json), asserting the renderer runs without throwing and includes every keyframe name.
3. Running "npm test."
4. Generating DESIGN-shopify.md from captures/shopify-winter2026.json into examples/ as a demonstration artifact.
5. Coding strictly against docs/capture-schema.md and verifying against the two real captures.
