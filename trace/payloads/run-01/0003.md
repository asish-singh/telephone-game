You are building the extraction engine of a CLI tool called "designdna" in the current directory (a fresh Node.js project, create package.json).

Goal: given a URL, render it in headless Chromium with Playwright and produce a single structured JSON file (the "design capture") describing the site's visual design exhaustively enough that another program can generate a replication guide from it.

Build:
1. package.json with playwright as dependency, run `npx playwright install chromium` so it works.
2. src/extract.js exporting `extractDesign(url) -> capture object`, and bin/designdna.js as CLI: `node bin/designdna.js <url> --json capture.json`.
3. The capture object must include:
   - meta: url, title, viewport used, timestamp
   - colors: frequency ranked list of all colors from computed styles, each with role hints (text, background, border, accent) and example selectors
   - typography: every font family in use, and for each a list of (size, weight, lineHeight, letterSpacing, example element/selector, sample text)
   - spacing: common margin/padding values ranked, container max widths, border radii, box shadows
   - breakpoints: from all stylesheets' media queries
   - animations: EVERY @keyframes rule found in all reachable stylesheets (verbatim CSS text), plus all transitions from computed styles (selector, properties, duration, easing), plus which elements use each keyframe animation. Note likely triggers (load, hover via :hover rules, scroll via IntersectionObserver hints or animation-timeline) where detectable.
   - backgrounds: for major sections, computed background (colors, gradients verbatim, image URLs), blend modes, filters, backdrop filters; detect video/canvas/WebGL hero elements
   - inlineSvgs: inline <svg> elements under 4KB verbatim, larger ones summarized (id/class, size, role)
   - sections: top level page outline (header, nav, hero, each major section, footer) with tag, classes, approximate height, key computed styles, and a one line structural description
4. Handle cross origin stylesheets gracefully (fetch their text when possible, skip with a note when not).
5. Write docs/capture-schema.md documenting the exact JSON shape you emit (another developer codes against this).
6. Test against https://www.shopify.com/editions/winter2026 and also a simple site like https://example.com. Commit nothing; just leave working code. Print a short report of what the shopify capture contained (counts of colors, keyframes, sections) at the end.

Constraints: plain Node.js (ES modules), no TypeScript, no framework beyond Playwright. Keep it robust; a failed sub extraction must not crash the run, record an error note in that field instead.
