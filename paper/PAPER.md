# The Telephone Game

**What happens to an instruction as it travels from a human, through an orchestrating AI, to coding subagents, and into code. A traced case study.**

Asish Singh, July 2026

## Abstract

One real software task was passed down a chain, a human wrote an intent message, a supervising model (Claude) decomposed it into instructions for two coding subagents (Codex), and the subagents implemented it under review. Every message between the layers was logged, 18 signals with verbatim payloads. Afterwards, each layer's understanding of the task was captured with an identical six question form and compared, requirement by requirement, by a neutral model that took no part in the build. Of the 9 requirements in the original message, 7 survived every layer with their meaning intact. Both mutations happened at the first hop, where the supervisor translated intent into instructions, and neither was ever revisited. Separately, three defects appeared during implementation that the workers' own tests did not catch, and all three were caught and corrected by the supervisor's independent acceptance runs. The case study suggests two practical lessons. The riskiest link in a delegation chain is the first translation, not the workers. And review earns its cost through defects that unit tests structurally cannot see.

## 1. Setup

The chain had four layers.

- **Layer 0, the human.** Wrote the intent in his own words, cold, without coaching.
- **Layer 1, the supervisor.** Claude decomposed the intent, issued instructions, reviewed returned work, and ran acceptance tests. By protocol it never wrote a line of the product's code.
- **Layer 2, the workers.** Two Codex CLI subagents, each seeing only its own instruction.
- **Layer 3, the code.** The merged, accepted implementation.

The task, chosen by the human, was a real product. A command line tool, designdna, that takes a URL, renders it in a headless browser, and writes an exhaustive machine readable DESIGN.md describing the site's colors, typography, layout, backgrounds, and every CSS keyframe animation verbatim, detailed enough for an AI agent to replicate the design. The tool was built, works, and lives in this repository under `build/designdna`.

## 2. Method

Two instruments, both in this repository.

**The trace.** Every signal between layers was logged as it happened to an append only JSONL file (`trace/run-01.jsonl`), with the full text of each message preserved verbatim (`trace/payloads/run-01/`). Signal types cover intent, decomposition, instruction, artifact, review, correction, resubmission, acceptance, and handoff. The logger and its validator are in `tools/trace.py`. Nothing in any figure is illustrative, every arrow, tick, and strand is generated by script from this file.

**The snapshots.** After the build, the same six question form (what is being built, required behaviors, constraints, edge cases, out of scope, how success is checked) was answered from each layer's viewpoint using only the material that layer received. S0 from the intent message, S1 from the issued instructions, S2 by each Codex agent from its own instruction, S3 reverse engineered from the final code alone by a model reading it cold. A neutral model (one that took no part in the build) then classified every S0 requirement at each later layer as preserved, mutated, or dropped, and linked every correction signal to the requirement it protected. Prompts are in `viz/prompts/`, results in `trace/analysis/run-01/`.

## 3. Findings

### 3.1 What survived

All substantive content requirements survived every layer intact, URL input, rendering the live site and reading its CSS and code, exhaustive replicate grade detail, theme and colors, animations and backgrounds, inline encoded elements, and the user's example page as the test case. The core of the ask, including its self declared "novel innovation" (animations described in enough detail for an agent to replicate), reached the code unharmed.

### 3.2 Where it drifted

Both mutations happened at the same place, the first hop, and neither was ever caught.

**R1, the platform.** The human asked for "a web or desktop app, whatever feasible". The supervisor's decomposition chose a command line tool, noted the choice once in an internal document, and never surfaced it back to the human as a question. Every layer downstream inherited the narrowing. The final product is a CLI. Arguably a reasonable engineering call, and the intent did say "whatever feasible", but the study's point stands, the choice was made silently at the translation step, and by layer 3 no trace of the original openness remained.

**R4, the filename.** The human wrote "design.md". From the first instruction onward it became "DESIGN.md". Nobody noticed until the neutral analysis. Harmless here, but a perfect miniature, literal details mutate at translation and then look official forever after.

The analysis also counted 16 downstream items with no antecedent in the intent (a JSON schema document, a test suite, numeric limits, a fixed viewport). Most are healthy elaboration that any builder must add to a two sentence ask. A few, like the fixed capture viewport, are genuinely new constraints a replicating agent would treat as the user's wish. Invention, not just loss, is part of drift.

### 3.3 What the feedback loop caught

Three defects reached the supervisor's acceptance runs despite the workers' tests passing. Each produced a correction loop, visible in the trace and the figures.

1. **The lost animations (seq 5 to 7).** The extractor's homemade CSS parser choked on the target page's minified stylesheet and silently dropped every keyframe, zero captured, while 40 named animations were demonstrably running on the page. The worker's unit tests passed, they tested the parser against CSS the parser could parse. The supervisor's acceptance run compared the capture against the live page and rejected. The rewritten parser recovered all 14 keyframe definitions with every use matched.
2. **The regression (seq 8 to 10).** The fix for an unrelated defect (a 220 entry "outline" collapsed to 17) introduced a new one, minimal pages now returned an empty outline. Corrections cause defects too, review has to re-run everything.
3. **The cross agent defect (seq 14 to 16).** Reviewing the second worker's output (the report writer) exposed a defect in the first worker's code, invisible elements like `head` and `meta` cited as color evidence. The defect surfaced one integration step downstream of where it lived.

The common thread, all three were invisible to the layer that created them and obvious one layer up. A worker's tests encode the worker's understanding, so they cannot catch the cases where that understanding is the problem.

## 5. Related work

The data structure behind this study, a temporal graph of spawn, delegate, communicate, and aggregate events, is what recent work calls an orchestration trace ([arXiv 2605.02801](https://arxiv.org/abs/2605.02801)). A 2026 survey formalizes the coordination topologies such systems use ([MDPI, Future Internet](https://doi.org/10.3390/fi18060326)), and GoAgent studies generating those topologies dynamically ([arXiv 2603.19677](https://arxiv.org/pdf/2603.19677)). The brain styled presentation follows a genuine research thread, BIGMAS organizes LLM agents as a brain inspired graph ([arXiv 2603.15371](https://arxiv.org/html/2603.15371v1)), and a brain inspired planning architecture appeared in Nature Communications ([s41467-025-63804-5](https://www.nature.com/articles/s41467-025-63804-5)). Those works evaluate on benchmarks. This study contributes something smaller and different, a fully traced, fully published single run of a real production style task, with the drift measured.

## 6. Limitations

This is one run of one task with one supervisor and two workers. Nothing here generalizes statistically. The snapshot method depends on a neutral model's judgment, the prompts are published so the judgment can be re-run. The supervisor that caused the R1 mutation also designed this study, the neutral classification was delegated precisely to limit that conflict, and the finding most unflattering to the supervisor is the paper's headline. The build task's code (designdna) is included, so every claim about defects and fixes can be checked against the diffs.

## 7. Practical takeaways

- The first translation of intent is the highest risk hop. If you delegate, make the translator surface its silent choices back to you as questions, "I chose a CLI, object?" costs one line.
- Workers' green tests are evidence about the workers' understanding, not about your intent. Independent acceptance runs at the layer above caught all three real defects here.
- Expect invention as well as loss. Downstream layers add constraints that look like your wishes. If an artifact will be consumed by another agent, mark what came from the user and what was assumed.
